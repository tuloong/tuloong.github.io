# 主页内容编写规范

本仓库是个人主页。更新 `index.html` 时，优先保持页面现有的朴素结构、排版和语言风格；除非明确要求，不做视觉重构或新增设计系统。

## 内容边界

- 不删除或改写用户已写的内容。既有文本是语气、长度和结构的参考。
- 新增内容以学习、阅读、生活和个人记录为主；不主动加入项目进展、工作汇报、代理治理或泛化的职业叙事。
- 不要写成宣传文、周报、学习报告或 AI 式鼓励语。语气应自然、直接，符合工程师的日常记录。
- 新增内容避免使用“自己”等刻意的第一人称自我指涉；不为普通记录添加“给自己的一句话”之类的标签。

## 学习条目格式

新增“目前在做”的学习事项，沿用已有条目结构：

```html
<n4><b> 学习主题 <a href="资料的官方或权威链接">资料名称</a>：</b>
    时间范围、明确目标与训练方向。 </n4>
```

- 提供具体、可访问的资料链接，优先官方、权威或原始来源。
- 明确目标方向，尽量包含可衡量的信息，例如周期、目标分数、每周数量或训练范围。
- 链接和目标必须来自用户提供的资料、已有记录或已验证来源；不要编造链接或学习目标。

## 动态新闻

- 动态新闻用简短的日记式记录，包含日期。
- 新增动态应与“目前在做”的学习方向一致，但避免重复整段计划说明。
- 更新动态后，同步修改页面末尾的“最后更新日期”。

## 修改与验证

- 只修改完成当前内容更新所需的区域。
- 保留仓库的换行符与既有 HTML 写法。
- 修改后至少检查 HTML 可被解析，并核对 diff 确认既有内容未被误删。
